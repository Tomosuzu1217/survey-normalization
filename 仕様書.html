アンケート分析ダッシュボード 仕様書

最終更新: 2026-03-05

1. 概要

ハイクラス・エグゼクティブ向け転職サービスに関するアンケート調査データを分析する、ブラウザ完結型シングルページダッシュボードです。

  • ファイル構成: index.html 1ファイル（HTML + CSS + JS を内包）
  • 外部ライブラリ: TailwindCSS (CDN), Chart.js + chartjs-plugin-datalabels (CDN), SheetJS / XLSX (CDN)
  • データ保存先: ブラウザのローカルストレージ（外部サーバーへの通信なし）

2. 入力ファイル

形式拡張子エンコーディング
Excel.xlsx / .xls自動
CSV.csvUTF-8（BOM付含む）/ Shift-JIS 自動判定

アップロード方法: ドラッグ＆ドロップ、またはファイル選択ダイアログ

3. 分析タブ（広報・IR / MP）

アップロード時に全行を2つのフィルタ条件で同時抽出し、タブごとに別々のデータとして保存します。

タブ年齢 (AGEID)年収 (SQ3)転職意向 (SQ5)
広報・IR4〜11（25〜64歳）5〜17（600万+）1〜6（意向者+経験者）
MP4〜9（25〜54歳）7〜17（800万+）1〜5（意向者のみ）

4. データ保存仕様

  • 保存先: localStorage
  • キー形式: survey3_{ir|mp}_{YYYY-MM}
  • 1レコードに含まれるデータ:
    • respondentCount — フィルタ通過者数
    • q1, q2 — 想起集計（s1 / total の辞書）
    • focus.services — サービス別詳細（想起順位、Q3-Q8評価、チャネル等）
    • wbQ1, wbQ2, wbN — WB加重集計
    • demo — 属性別クロス集計（性別/年齢/年収/役職/転職意向 × Q1/Q2 想起）
  • 旧キー（survey3_YYYY-MM）は起動時に自動削除

5. 表示機能一覧

5.1 自社サービス深掘り分析

JAC Recruitment を固定対象として、全設問の結果を横断表示します。

  • KPIメトリクスカード（8指標）: Q1想起人数、Q1第一想起率、Q2想起人数、Q2第一想起率、設問認知率、利用意向率、両領域想起率、平均スコア
  • 順位別想起分布チャート
  • 併記されやすいサービス Top10
  • ファネル指標（認知→関心→利用意向）
  • 接触チャネル Top10
  • 認知・関心・利用意向の分布チャート
  • Q1×Q2 クロス集計テーブル
  • 併記クロス集計テーブル
  • 設問ベース評価サマリーテーブル
  • 月別比較テーブル

5.2 Q1 / Q2 想起チャート

  • 第一想起（S1）Top10 — 横棒グラフ（%表示）
  • 全体想起（S1〜S5）Top10 — 横棒グラフ（%表示）
  • サマリーカード: N数、第一想起トップ(%)、全体想起トップ(%)、第一想起率
  • 詳細データテーブル（全サービス、S1/合計 × n(%)形式）

5.3 WB加重チャート WB

  • Q1・Q2それぞれに第一想起/全体想起のWB加重版チャート（アンバー色）
  • WB係数で重み付けした割合を表示

5.4 属性別クロス集計 NEW

  • Q1・Q2カード末尾のアコーディオン（<details>）で展開
  • 5属性の切替タブ: 性別 / 年齢 / 年収 / 役職 / 転職意向
  • 属性カテゴリ別のグループ横棒グラフ（TOP5サービス × 属性カテゴリ、%表示）
    • 第一想起（S1）属性別
    • 全体想起 属性別
  • クロス集計テーブル: 属性カテゴリ × TOP10サービス（n(%)形式）
  • 遅延レンダリング: アコーディオン展開時にのみ描画（パフォーマンス配慮）

5.6 月次トレンド

  • 2か月分以上のデータがあると自動表示
  • JAC Recruitment の月別推移: 想起・第一想起・認知・利用意向
  • 折れ線チャート + テーブル

5.7 複数月比較モード

  • 比較月チップをクリックして複数月を選択
  • Q1/Q2チャートが折れ線グラフに切り替わり、サービス別の時系列推移を表示
  • 比較テーブルで月別のS1/合計を一覧表示

6. サービス名正規化

自由記述のサービス名を SERVICE_DICT で統一名に正規化します。

  • 全角/半角統一、スペース除去、大文字化
  • 完全一致 → 部分一致 → 正規表現マッチ の順で照合
  • 未マッチの場合は入力値をそのまま使用
  • 「特にない」「なし」「思いつかない」等は除外

7. CSV エンコーディング検出

  1. BOM検出（UTF-8 BOM: EF BB BF、UTF-16 LE/BE）
  2. UTF-8 マルチバイトシーケンスの完全検証（最大8KB）
  3. Shift-JIS パターン検出（2バイト文字の先頭バイト 0x81-0x9F, 0xE0-0xFC）
  4. デフォルト: UTF-8

8. 技術仕様

項目詳細
ファイル構成シングルHTML（インラインCSS + JS）
CSSTailwindCSS 3.x (CDN)
チャートChart.js 4.x + chartjs-plugin-datalabels 2.x
ファイル読込SheetJS (xlsx.full.min.js) CDN
データ保存localStorage（キー: survey3_{tab}_{month}）
非同期処理バッチ処理 + setTimeout(0) でUIブロッキング回避
チャート管理グローバル CHARTS オブジェクトで destroy/recreate
プログレスアップロード時にプログレスバー表示（IR集計→MP集計→保存）

9. 制約・注意事項

  • データはブラウザのローカルストレージに保存されるため、他のブラウザ・デバイスとは共有されません
  • ローカルストレージの容量制限（通常5〜10MB）を超えると保存に失敗します
  • 同じ年月のデータを再アップロードすると上書きされます
  • 属性別クロス集計は新規アップロード分から有効です（旧データには demo フィールドがないため表示されません）
  • 外部サーバーへの通信は一切行いません（完全オフライン動作）
